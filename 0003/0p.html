Surds_(0001). (9979). 

In mathematics, a Surd is an expression, containing a root term , with an irrational value, that can not be expressed exactly, for example(0002).

the square root of 3 = 1.732050808(0003).

 

Sometimes it is useful to work in surds, rather than using an approximate decimal value(0004).Surds can be manipulated just like algebraic expressions, and sometimes it may be possible to eliminate the surd term, _called rationalizing the expression_ ,which may have not been possible if you tried to work with the approximate value(0005). When asked to give the exact value, approximate decimal answers will not do, and you will have to manipulate surds in order to give a final answer in simplified surd form(0006).

Simplification of surds(0007). 

Because surds can be manipulated like algebraic expressions, you can easily multiply out the terms, and add the like terms(0008).However, there are also a few rules that will be useful when simplifying surds(0009).

b0003 

A Basic rule of surds(0010). 

Because_,

root x times root x = x(0011). 

It can be rearranged to give

root x = x over, root x(0012). 

and,

1 over, root x = root x over, x(0013).

Surds as indices(0014). 

Because__,

the n, root of x = x by 1 over, n(0015).

the laws of indices, therefore, also apply to any n, root(0016).The most frequently used instances of this are laws 4, and 5 with surds:(0017).

since,

x, y, by n, = x by n, y by n(0018).  

this is equivalent to,

root x, y, = root x times root y(0019).

x over, y by n, = x by n, over, y by n(0020). 

becomes,

root, x over, y = root x over, root y(0021).

The first of these points is often used to simplify a surd, for example(0022). 

root 200 = root 100 times 2 = root 100 time root 2 = 10 root 2(0023). 

In an exam, you will be expected to write all surds with the smallest possible number inside the surd, the number inside the root shouldn t have any square factors(0024).

b0009 

Rationalizing the denominator(0027). 

Another technique to simplify expressions involving surds is to rationalize the denominator(0028).This means removing a surd from a fraction(0029). 

In the case of a fraction such as,

5 over, root 3(0030). 

both the numerator, and the denominator can be multiplied by,

root 3, to give 5 root 3 over, 3(0031). 

b0010 

If the fraction is of the form,

A, over, b plus root c(0032). 

b0011 

the strategy used in the previous paragraph will work, if it is slightly modified(0033). 

This time you should multiply the numerator, and denominator by,

b minus root c(0034).

b0012 

If you are familiar with the standard difference of two squares expansion, you should already know what happens next:(0035). 

'we get,

A over, b plus root c, times b minus root c over, b minus root c = A b, minus A root c over, b squared plus b root c minus b root c minus root c squared = A b, minus A root c over, b squared minus c(0036). 

As you can see the denominator now does not contain any surds(0037).

b0013 

For example,

2 over, root 3 minus 1 times root 3 plus 1 over, root 3 plus 1 = 2 root 3 plus 1 over, 3 minus 1 = root 3 plus 1(0038).

Common questions, and mistakes(0039). 

Splitting up roots sums instead of products(0040). 

A common mistake is to split,

 

root x plus y into root x plus root y, or x plus y squared into x squared plus y squared(0041). usually whilst moving it to the other side of the equals(0042).

b0015 

Trying a few examples, will quickly convince you that, this is not possible: 

root 64 is not equal to root 32 plus root 32 and root 25 is not equal to root 12 plus root 13, and so on(0043).
